## Purpose

This repo holds the canonical registry of all external service accounts that are used across the different entities (weibeld, W5D Labs, Rye & Cheese, w5d.io). It serves two purposes:

  1. A human-readable overview of all accounts and services in one place
  2. A programmatic reference for platform systems that need non-secret identifiers
     (e.g. a Terraform config for VPN infrastructure reading the AWS account ID from here)

This repo is consumed by other repos in the platform (e.g. 'presence') as a cross-repo reference. Secrets (API keys, passwords) are never stored here — only non-secret identifiers.


## Current state

accounts.yaml holds a map of service name → account details. Currently only login emails and account names are recorded. Example entry:

  cloudflare:
    email: danielmweibel@gmail.com
    account-name: w5dio


## What needs to be done

Extend accounts.yaml to include non-secret account identifiers for services that have them. These are stable identifiers required for programmatic access (API calls, Terraform, CLI) that are safe to store in version control. Examples:

  - Cloudflare: account_id (required by Terraform provider)
  - AWS: account ID (12-digit number, needed in IAM policies and Terraform)
  - GCP: project ID / project number
  - Porkbun / Namecheap: API-level account username if different from login

Not all YAML fields apply to all services. Omit fields that don't exist or aren't relevant.
